Process_Step:
Process_Description:
AutoCAD 2007 was used with bearing and distance calls or northing and easting points where available. After a thorough research was completed to assure that the most accurate source was used, legal descriptions and property deeds were interpreted and metes and bounds entered into an AutoCAD DWG file. The AutoCAD cogo file was created using a projected USGS DLG quad file in Florida State Plane NAD83. Once the cogo file was completed, cleaned and edited, using ArcGIS 9.2 it was imported into the individual parks' File Geodatabase. Then the correct parcel attribution data is added to the File Geodatabase. Subsequently, a shapefile, in the Fl DEP Albers coordinate system, was created from the File Geodatabase. Lastly, the new statewide shapefile called FSP.shp is created bi-annually using ArcGIS 9.2 to merge together all the individual park boundary shapefiles.
